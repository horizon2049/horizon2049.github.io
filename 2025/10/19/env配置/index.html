env配置

Vite 在一个特殊的 import.meta.env 对象上暴露环境变量

  • import.meta.env.MODE: {string} 应用运行的模式。
  • import.meta.env.BASE_URL: {string} 部署应用时的基本 URL。他由base 配置项决定。
  • import.meta.env.PROD: {boolean} 应用是否运行在生产环境（使用 NODE_ENV='production' 运行开发服务器或构建应用时使用 NODE_ENV='production' ）。
  • import.meta.env.DEV: {boolean} 应用是否运行在开发环境 (永远与 import.meta.env.PROD相反)。
  • import.meta.env.SSR: {boolean} 应用是否运行在 server 上。

Cross-env使用

Cross-env 是一个跨平台设置环境变量的工具，它解决了在不同操作系统上设置环境变量的兼容性问题。在 Windows 和 POSIX（Linux、macOS）系统上，设置环境变量的语法不同，cross-env 提供了一致的解决方案

  • 在 POSIX 系统上：NODE_ENV=production node app.js
  • 在 Windows 上：SET NODE_ENV=production && node app.js

cross-env 就是为解决这个问题而设计的
